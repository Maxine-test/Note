GitHub 公開網頁 × Google Sheet 當資料庫

操作筆記 · 給靜態 HTML（如網球紀錄）用 GitHub Pages 發佈，並用 Apps Script 讀寫試算表

一、把做好的 HTML 用 GitHub 公開給任何人讀取

目標：讓任何人用瀏覽器開網址就能看到你的頁面，不需登入 GitHub。

1.1 準備 GitHub 帳號與新儲存庫

  1. 到 github.com 註冊或登入。
  2. 右上角 ＋ → New repository。
  3. 取專案名稱（例如 tennis-tracker），選 Public（公開），建立儲存庫。

1.2 上傳 HTML 檔案

擇一即可：

  • 網頁上傳：在 repo 頁面 Add file → Upload files，拖入 .html，填 commit 說明後 Commit changes。
  • 本機 Git：在專案資料夾執行 git init、git remote add origin …、git add .、git commit、git push -u origin main。

1.3 開啟 GitHub Pages 並選對資料夾（重要）

  1. 進入該 repo → Settings → 左側 Pages（在「Code and automation」底下）。
  2. Build and deployment → Source 選 Deploy from a branch。
  3. Branch 選 main（或你實際使用的預設分支）。
  4. Folder 二選一，必須和「檔案實際位置」一致：
    Folder 設定你的 HTML 要放在 repo 哪裡
    / (root)與 README 同層，在根目錄
    /docs必須有 docs 資料夾，HTML 放在 docs/ 裡
  5. 按 Save。若選了 /docs 但 repo 沒有 docs 資料夾，Actions 裡 Jekyll 建置會失敗（commit 旁出現紅色 ✕）。

1.4 等建置完成並取得網址

  1. 開啟 repo 的 Actions 分頁，等 pages build and deployment 出現綠色勾勾（或至少不再失敗）。
  2. 回到 Settings → Pages，上方會顯示網站網址，格式通常為：
    https://你的帳號.github.io/儲存庫名稱/
  3. 若檔名不是 index.html，請在網址後加上檔名，例如：
    https://你的帳號.github.io/tennis-tracker/tracker-sheet.html

常見問題

  • 404 或舊畫面：確認 Folder 與檔案路徑一致；建置完成後等 1～2 分鐘再重新整理，必要時強制重新整理（Ctrl+F5）。
  • 紅色 ✕：多半是 /docs 不存在或為空，改為 / (root) 或建立 docs 並把 HTML 移入。

二、用 Google 試算表當「免費雲端資料庫」

靜態網頁無法直接寫入試算表，需透過 Google Apps Script 部署成 Web App，由網頁用 fetch 呼叫該網址來讀寫。

2.1 兩種常見後端模式（先選一種）

模式試算表長相適用
A · 不自訂格式整包 JSON 存在單一儲存格（例如分頁 AppData 的 A1），不在試算表裡「一格一格欄位」維護最快上手、與網頁資料結構 1:1
B · 自訂格式除 A1 整包備份外，另寫入 records、settings 等分頁，有表頭與多列（方便人用眼睛看、篩選）要報表、要與 tracker-sheet.html 的 sheetModel 對齊

以下 程式碼.gs 範例皆需刪除 Google 預設的 function myFunction() {}，整檔貼上後再部署。

2.2 建立試算表與 Apps Script

  1. 開啟 Google 試算表，新增試算表（建議從這張表按 擴充功能 → Apps Script，腳本才會綁定到正確的 SpreadsheetApp.getActiveSpreadsheet()）。
  2. 在編輯器開啟 程式碼.gs，貼上「模式 A」或「模式 B」其中一份完整程式碼。
  3. 按 Ctrl+S 儲存；可選：上方選 doGet 按 執行 完成一次授權。

2.3 模式 A：新增後端 · 不自訂試算表欄位（整包 JSON）

行為：doPost 把網頁送來的 JSON 字串整包寫入 AppData!A1；doGet 讀出同一字串回傳。清空時網頁送本文 null 會清掉 AppData 分頁內容。

若要改存別的儲存格：修改常數 DATA_SHEET、DATA_CELL。

const DATA_SHEET = 'AppData';
const DATA_CELL = 'A1';

function doGet() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sh = ss.getSheetByName(DATA_SHEET);
  if (!sh) {
    return ContentService.createTextOutput('null').setMimeType(ContentService.MimeType.JSON);
  }
  const raw = sh.getRange(DATA_CELL).getValue();
  if (raw === '' || raw === null || raw === undefined) {
    return ContentService.createTextOutput('null').setMimeType(ContentService.MimeType.JSON);
  }
  const str = typeof raw === 'string' ? raw : JSON.stringify(raw);
  try {
    JSON.parse(str);
  } catch (e) {
    return ContentService.createTextOutput(JSON.stringify({ error: 'invalid_json_in_sheet' }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  return ContentService.createTextOutput(str).setMimeType(ContentService.MimeType.JSON);
}

function doPost(e) {
  const body = e.postData ? e.postData.contents : '';
  if (body === null || body === undefined || String(body).trim() === '' || String(body).trim() === 'null') {
    clearAppData_();
    return ContentService.createTextOutput(JSON.stringify({ ok: true, cleared: true }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  let obj;
  try {
    obj = JSON.parse(body);
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({ ok: false, error: 'bad_json' }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let sh = ss.getSheetByName(DATA_SHEET);
  if (!sh) sh = ss.insertSheet(DATA_SHEET);
  sh.getRange(DATA_CELL).setValue(JSON.stringify(obj));
  return ContentService.createTextOutput(JSON.stringify({ ok: true }))
    .setMimeType(ContentService.MimeType.JSON);
}

function clearAppData_() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sh = ss.getSheetByName(DATA_SHEET);
  if (sh) sh.clear();
}

2.4 模式 B：新增後端 · 自訂分頁與欄位（records / settings）

在模式 A 的基礎上，若 POST 的 JSON 裡帶有 sheetModel.sheets（例如 tracker-sheet.html 的 save()），會另外寫入：

  • records：表頭 id, fullDate, dateLabel, type, hours, courtNote, classNote, phaseStart, createdAt，以下一列一筆紀錄。
  • settings：表頭 key, value，列如 currentPhaseStart、phases（JSON 字串）、courtDefault、remarkSchedule（JSON 字串）、maxHours。

若要改分頁名稱或欄位：須同時改 Apps Script 的 writeSheetModel_ / writeTableSheet_，以及網頁端產生 sheetModel 的程式（見下節 HTML）。

const DATA_SHEET = 'AppData';
const DATA_CELL = 'A1';

function doGet() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sh = ss.getSheetByName(DATA_SHEET);
  if (!sh) {
    return ContentService.createTextOutput('null').setMimeType(ContentService.MimeType.JSON);
  }
  const raw = sh.getRange(DATA_CELL).getValue();
  if (raw === '' || raw === null || raw === undefined) {
    return ContentService.createTextOutput('null').setMimeType(ContentService.MimeType.JSON);
  }
  const str = typeof raw === 'string' ? raw : JSON.stringify(raw);
  try {
    JSON.parse(str);
  } catch (e) {
    return ContentService.createTextOutput(JSON.stringify({ error: 'invalid_json_in_sheet' }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  return ContentService.createTextOutput(str).setMimeType(ContentService.MimeType.JSON);
}

function doPost(e) {
  const body = e.postData ? e.postData.contents : '';
  if (body === null || body === undefined || String(body).trim() === '' || String(body).trim() === 'null') {
    clearAll_();
    return ContentService.createTextOutput(JSON.stringify({ ok: true, cleared: true }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  let obj;
  try {
    obj = JSON.parse(body);
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({ ok: false, error: 'bad_json' }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  savePayload_(obj);
  return ContentService.createTextOutput(JSON.stringify({ ok: true }))
    .setMimeType(ContentService.MimeType.JSON);
}

function savePayload_(obj) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  let sh = ss.getSheetByName(DATA_SHEET);
  if (!sh) sh = ss.insertSheet(DATA_SHEET);
  sh.getRange(DATA_CELL).setValue(JSON.stringify(obj));
  if (obj && obj.sheetModel && obj.sheetModel.sheets) {
    writeSheetModel_(ss, obj.sheetModel);
  }
}

function writeSheetModel_(ss, sheetModel) {
  const sm = sheetModel.sheets;
  if (sm.records) writeTableSheet_(ss, 'records', sm.records.headers, sm.records.rows);
  if (sm.settings) writeTableSheet_(ss, 'settings', sm.settings.headers, sm.settings.rows);
}

function writeTableSheet_(ss, name, headers, rows) {
  if (!headers || !headers.length) return;
  let sh = ss.getSheetByName(name);
  if (!sh) sh = ss.insertSheet(name);
  sh.clear();
  const data = [headers].concat(rows || []);
  sh.getRange(1, 1, data.length, headers.length).setValues(data);
}

function clearAll_() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  ['AppData', 'records', 'settings'].forEach(function (n) {
    const sh = ss.getSheetByName(n);
    if (sh) sh.clear();
  });
}

2.5 修改試算表 Web App 網址（tracker-sheet.html）

檔案：tracker-sheet.html（本機路徑例如 OneDrive/AI/tracker-sheet.html，或你已上傳到 GitHub 的那份）。

重點：之後若要換網址，主要改「一個常數」

在 <script> 開頭設定區，找到 DEFAULT_WEBAPP_URL，只改這一行引號裡的網址即可當作全站／全檔預設的 Web App。其餘 getWebAppUrl()、load()、syncPostSheet() 會自動使用它（除非瀏覽器另有自訂覆寫）。

2.5.1 具體怎麼改（逐步）

  1. 用文字編輯器或 Cursor 開啟 tracker-sheet.html。
  2. 按 Ctrl+F（Mac：Cmd+F）搜尋：DEFAULT_WEBAPP_URL。
  3. 會看到類似下面這一行（僅示意，你的網址會不同）：
const DEFAULT_WEBAPP_URL = 'https://script.google.com/macros/s/你的部署ID/exec';
  1. 把兩個單引號中間整串網址，換成你在 Apps Script 部署後複製的新網址（必須以 /exec 結尾，通常長度很長屬正常）。
  2. 不要刪掉兩端的單引號 '，也不要在網址中間斷行。
  3. 存檔。
  4. 若網頁是放在 GitHub Pages：把更新後的 tracker-sheet.html 再 commit / push（或網頁上傳覆蓋），等 Actions 綠勾後重新整理頁面。
  5. 若你只在本機雙擊開啟 HTML：存檔後重新整理瀏覽器即可。

2.5.2 不需要為了換網址而改的東西

  • 一般情況不必改 WEBAPP_URL_KEY（那是 localStorage 的鍵名，改掉會讓舊的自訂網址讀不到）。
  • 不必改 getWebAppUrl、syncPostSheet 函式本體，除非你要自己改邏輯。

2.5.3 與「檔案預設」並存：瀏覽器自訂網址（選用）

要做的事改哪裡怎麼改
預設給所有人同一個 /exec（推薦用常數）DEFAULT_WEBAPP_URL見上方 2.5.1
只在某台電腦暫時換網址（不動檔案、不上傳 GitHub）網頁按 ☁️ 雲端貼上網址後確定；會寫入 localStorage 鍵 tennis_sheet_webapp_url，優先於 DEFAULT_WEBAPP_URL
清除瀏覽器自訂、改回檔案裡的預設常數☁️ 雲端輸入框留空 → 確定 → 會 removeItem('tennis_sheet_webapp_url')

相關程式（想進階再改時，用搜尋跳轉）：WEBAPP_URL_KEY、DEFAULT_WEBAPP_URL、function getWebAppUrl()、function syncPostSheet、function load()、openCloudUrlSettings。

若你自寫的 HTML 沒有 DEFAULT_WEBAPP_URL：在 <script> 設定區把 fetch 用的網址變數（舊名可能叫 DB_URL）改成你的 /exec；POST 請用 Content-Type: text/plain;charset=utf-8，本文仍為 JSON 字串（見下節）。

2.6 前端與後端如何對接（POST / GET）

  • GET：fetch(你的/exec)，回應為 JSON 字串；全新試算表尚未寫入時，瀏覽器可能看到 null，屬正常。
  • POST：headers: { 'Content-Type': 'text/plain;charset=utf-8' }，body: JSON.stringify(物件)。不要用 application/json，以免觸發 CORS 預檢在部分環境失敗。
  • 清空：tracker-sheet.html 會 POST 本文 null；後端需依 2.3 / 2.4 範例清除對應分頁。

2.7 自訂格式時，網頁端要改哪裡（sheetModel）

若你使用 tracker-sheet.html：搜尋 function buildSheetModel()。要改的通常是：

  • recordsHeaders 陣列順序與名稱（須與 Apps Script 寫入的分頁欄位一致）。
  • recordsRows 的 map，讓每一列對應到 data.history 裡的欄位。
  • settingsRows：要同步到 settings 分頁的 key/value。

改完網頁後，Apps Script 模式 B 的 writeSheetModel_ 分頁名稱若與網頁不一致，也要一併修改。

2.8 部署為 Web 應用程式

  1. Apps Script 編輯器右上角 部署 → 新增部署作業。
  2. 類型選 網路應用程式。
  3. 執行身分：選「我」。
  4. 具有存取權的使用者：從 GitHub Pages 匿名讀寫請選 任何人。首次部署會要求授權。
  5. 複製 網路應用程式 URL（結尾 /exec），貼到 tracker-sheet.html 的 DEFAULT_WEBAPP_URL 或 ☁️ 雲端設定。

2.9 修改腳本後要「重新部署」

  1. 部署 → 管理部署作業 → 選既有 Web App → 編輯（鉛筆）。
  2. 版本 選 新版本，再按部署。僅儲存程式碼不會更新線上行為。

安全與限度（必讀）

  • Web App 網址若被取得，他人也可能呼叫；不適合存放密碼、個資、金流。僅適合低風險、可公開或半公開的資料。
  • Google 有每日執行時間與呼叫次數配額，大量使用者時可能觸發限制。
  • 試算表權限與「執行身分」會影響誰能寫入哪一格，部署前請再確認。
