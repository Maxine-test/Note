n8n 入門安裝與基本操作手冊

開源工作流自動化工具 · 安裝指南、介面說明與第一個工作流

一、什麼是 n8n？

n8n 是開源的工作流自動化（Workflow Automation）平台，可串接多種服務（如 Gmail、Slack、Google Sheets、資料庫、API 等），用視覺化節點與連線建立自動化流程。每個節點代表一個動作或觸發條件，節點之間以連線決定執行順序，無需寫太多程式即可完成整合與排程。

特點：自架可完全掌控資料、支援 400+ 節點與 1000+ 整合、可搭配 AI 節點、提供 Community 免費版與付費 Cloud／Enterprise 方案。

二、系統需求

  • Node.js（npm 安裝時）：20.19～24.x
  • Docker（容器安裝時）：20.10+
  • 生產環境：建議使用 PostgreSQL 13+ 取代預設 SQLite，並至少 8GB RAM；僅測試可用 2GB
  • 網路：需能連線以使用 Web 介面及對外 API／服務

三、安裝方式

3.1 快速試用（不安裝）

本機已安裝 Node.js 時，可直接執行：

npx n8n

啟動後在瀏覽器開啟 http://localhost:5678 即可使用。關閉終端機即停止，資料預設存於本機。

3.2 全域安裝（npm）

適合本機開發或小規模使用：

npm install n8n -g
n8n

或使用 n8n start 啟動。更新：npm update -g n8n。

3.3 Docker 安裝（建議生產環境）

基本單一容器：

docker volume create n8n_data
docker run -d --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n

存取一樣為 http://localhost:5678。正式環境建議用 Docker Compose 搭配 PostgreSQL，並設定 N8N_ENCRYPTION_KEY、N8N_WEBHOOK_URL、N8N_EDITOR_BASE_URL 等環境變數。

3.4 n8n Cloud（線上版，免安裝）

若不想管理主機或本機環境不符（例如 Node.js 過舊、未安裝 Docker），可直接使用線上版：

  • 前往 n8n Cloud 註冊帳號。
  • 在瀏覽器內即可建立、編輯、執行工作流，無需在本機安裝任何軟體。
  • 有免費方案可試用，進階方案依用量計費。

四、首次啟動與存取

  1. 依上述任一方式啟動 n8n（例如 npx n8n 或 n8n）。
  2. 瀏覽器開啟 http://localhost:5678。
  3. 首次使用會引導建立擁有者帳號（自架版可設定使用者名稱與密碼）。
  4. 登入後進入「工作流總覽」，可建立新工作流或開啟範本。

五、介面簡介

  • 左側：工作流列表、範本、設定等。
  • 中間畫布：編排節點（Node）、以連線決定執行順序。
  • 節點：每個方塊代表一個步驟（觸發、動作、邏輯等）；點節點可於右側編輯參數。
  • 右側：目前選取節點的設定與輸入／輸出預覽。
  • 上方工具列：儲存、執行工作流、切換啟用／停用等。

六、核心概念：節點、觸發器、工作流

6.1 節點（Node）

節點是工作流的基本單位，一個節點代表一個動作或功能，例如：發送郵件、寫入試算表、呼叫 API、條件分支、撰寫自訂程式碼等。n8n 內建數百個節點，涵蓋常見雲端服務與工具。

6.2 觸發器（Trigger）

觸發器是「起點」節點，決定工作流何時被執行。常見類型：

觸發器說明
Schedule Trigger / Cron依排程（每日、每小時、自訂 Cron）執行
Webhook收到 HTTP 請求時執行（表單、API 回呼等）
App 觸發（如 Gmail、Slack）當所連服務發生事件時執行
Manual Trigger手動按「執行」按鈕觸發

6.3 工作流（Workflow）

工作流 = 一組已連線的節點形成的自動化流程。例如：Webhook 收表單 → 寫入 Google Sheets → 用 Gmail 寄確認信。

七、基本操作：建立第一個工作流

  1. 在總覽頁點「新增工作流」或從左側進入空白工作流。
  2. 點「加入第一個步驟」，選擇一個觸發器（例如 Manual Trigger 或 Schedule Trigger）。
  3. 在觸發器右側點 +，加入下一個節點（例如 HTTP Request、Gmail、Google Sheets 等），並在節點內填寫所需參數（URL、認證等）。
  4. 用連線將觸發器與後續節點依序接起來（拖曳節點上的圓點連到下一節點）。
  5. 儲存工作流（上方儲存按鈕或快捷鍵）。
  6. 點「執行工作流」或「Test workflow」手動測試，在右側查看每個節點的輸入／輸出。
  7. 確認無誤後，若需自動執行，請將工作流啟用（Active）；排程或 Webhook 觸發才會依設定運行。

八、常用節點類型簡介

類型用途
Trigger 類排程、Webhook、各 App 事件
Application 類Gmail、Slack、Sheets、Notion、資料庫等
Core 類IF 條件、Switch、Merge、Split、Set 等邏輯與資料處理
Code / Function自訂 JavaScript 等程式邏輯

九、生產環境與安全建議

  • 加密金鑰：設定 N8N_ENCRYPTION_KEY（環境變數），金鑰遺失將無法解密已儲存的憑證。
  • 資料庫：正式環境建議改用 PostgreSQL，避免容器重啟導致 SQLite 資料問題。
  • 對外網址：若有 Webhook 或排程需從外網觸發，需設定 N8N_WEBHOOK_URL、N8N_EDITOR_BASE_URL，並建議搭配 HTTPS／反向代理。
  • 使用者與認證：自架版可設定多使用者與 SSO、2FA 等，詳見官方 Authentication 文件。

十、注意事項

  • 自架 n8n 需要一定的伺服器與維運經驗；設定錯誤可能導致資料遺失、資安風險或服務中斷。若不熟悉維運，可優先考慮 n8n Cloud。
  • 執行含敏感資料的工作流時，請注意憑證存放與網路傳輸安全（HTTPS、防火牆等）。

十一、從 JSON 匯入並啟用工作流

若你已有 n8n 匯出的工作流 JSON 檔，可依下列方式匯入並啟用。

11.1 在編輯器內匯入（圖形介面）

  1. 登入 n8n（自架或 n8n Cloud）。
  2. 點右上角 ⋯（三個點）選單。
  3. 選擇 Import from File，選取你的 .json 檔案；或選 Import from URL 貼上 JSON 的網址。
  4. 匯入後工作流會出現在畫布上，檢查節點與連線是否正確。
  5. 若工作流內有使用憑證（Gmail、API Key 等），需在對應節點重新設定一次（匯入不會帶入密碼）。
  6. 儲存後，若要自動執行，將右上角開關切為啟用（Active）即可。

11.2 貼上 JSON 內容

若不想用檔案：可開啟「新增工作流」→ 點選單 Import from File 旁若有「從剪貼簿貼上」或直接將 JSON 內容貼到可接受貼上的匯入欄位（依你使用的 n8n 版本介面為準）。

11.3 指令列匯入（自架版）

自架 n8n 且可連到主機指令列時，可在 n8n 安裝目錄執行：

n8n import:workflow --input=你的工作流.json

匯入後到網頁介面找到該工作流，補上憑證並儲存，再啟用即可。

注意：匯入前請確認 JSON 來自可信來源；若為他人分享的檔案，建議檢查並移除或改寫內含的帳密、API Key 等敏感資訊。
